# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repo state

This is a **greenfield** repo. No application code exists yet — only a build brief and a design-system handoff. The target application is a static landing page called **"The Bridge Insights"** (Astro 5 + Cloudflare Pages). The first task any Claude instance receives here will almost certainly be scaffolding Fase 1 from the brief.

Key inputs to read **before writing code**:

1. `INSIGHTS_BRIEF.md` — stack decisions, file-tree target, env vars, and phased prompts (Fase 1 → 4).
2. `_handoff/design-system/README.md` — brand context, content rules, visual foundations.
3. `_handoff/design-system/colors_and_type.css` — source-of-truth CSS vars for Tailwind theme.
4. `_handoff/design-system/prototype.html` — approved visual reference for the 7-section landing. Migrate component-by-component.
5. `_handoff/design-system/SKILL.md` — skill-style design guidance (same brand as production).

## Stack (locked)

Astro 5 (SSG) · Tailwind 3 · Cloudflare Pages · Cloudflare Pages Functions (`functions/api/subscribe.ts`) · Notion as CMS (build-time fetch, not runtime) · GitHub Actions cron for rebuilds · **pnpm**.

Explicitly **not** used: Next.js, Vercel, Docker, shadcn, Framer Motion, analytics, SSR, ISR. Motion is CSS-only (or Motion One if ever needed).

## Phased workflow

The brief defines four phases. Do not skip ahead. Each phase has a specific "devuélveme X antes de escribir código" contract — honour it.

- **Fase 1** — Scaffold Astro+Tailwind, migrate `prototype.html` to ~9 components, mock data in `src/data/mock.ts`. Must pass `pnpm build` with no warnings.
- **Fase 2** — Notion wiring. `scripts/fetch-notion.mjs` runs as `prebuild` and writes `src/data/*.json`. Components import JSON, never call Notion at request time. Fallback to `mock.ts` when `NOTION_TOKEN` is absent.
- **Fase 3** — `functions/api/subscribe.ts` Pages Function. In edge code use `fetch` against the Notion REST API, **not** `@notionhq/client`.
- **Fase 4** — Deploy to Cloudflare Pages + GitHub Actions cron (every 6 h) hitting the deploy-hook URL.

## Commands

Not installed yet — they come from Fase 1. Once `package.json` exists, expect:

- `pnpm dev` — Astro dev server
- `pnpm build` — runs `prebuild` (`node scripts/fetch-notion.mjs`) then `astro build`. **Zero-warning builds are a gate** between phases.
- `pnpm wrangler pages dev ./dist --compatibility-date=2025-04-01` — local Pages Functions testing (Fase 3+)

## Architecture notes

- **Build-time data only.** `src/data/*.json` is generated by `scripts/fetch-notion.mjs` from Notion and is `.gitignore`d. Components read those JSON files directly; they never import `@notionhq/client`.
- **Edge vs Node split.** Node-side Notion access (build-time) uses `@notionhq/client`. Edge-side Notion access (the subscribe Function) uses raw `fetch` — keep these separate, don't share a client module across both.
- **Tokens are the contract with design.** Tailwind `theme.extend` mirrors `_handoff/design-system/colors_and_type.css`. When a design token changes, change it in the CSS file first, then mirror into Tailwind config — don't invent new color/spacing values in components.
- **Fonts are local.** Futura Std Book + Medium ship from `public/fonts/` via `@font-face`. Do **not** attempt to load Futura from Google Fonts (it isn't there and the license is already satisfied by the local files). Inter Tight is the only Google-Fonts family.
- **Env vars** (see brief §3): `NOTION_TOKEN`, `NOTION_PUBLICATIONS_DB_ID`, `NOTION_BIWEEKLY_DB_ID`, `NOTION_SUBSCRIBERS_DB_ID`. DB IDs for Publications and Biweekly are pinned in the brief.

## Brand rules that affect code output

These are enforced in user-facing copy; breaking them requires re-doing work.

- **Language: Spanish (Castilian).** English only for role titles and proper nouns (`VanguardIA`, `TRAIN / PRACTICE / APPLY`).
- **Sentence case everywhere.** All-caps is reserved for the `TRAIN / PRACTICE / APPLY` triptych. No shouty headlines, no all-caps buttons.
- **No emoji.** Anywhere. Use real pictograms (Material Symbols Rounded 300 is the nearest match if needed).
- **Curly quotes `" "`, en dashes for ranges.** Not straight quotes, not hyphens.
- **Two-accent system, never mixed in one composition:** red `#ef3340` on light pages, mint `#73ffba` on dark pages. For the Insights landing (dark-first), expect mint as the live accent.
- **Stats are hero content.** Big number + short caption — e.g. `+10` / `Soluciones creadas`, `30hs` (no space) / `De aprendizaje aplicado`.
- **Respect `prefers-reduced-motion`** for the CSS fade-up-on-scroll; motion spec is ≤12px rise, 200–280ms, `cubic-bezier(0.2, 0.8, 0.2, 1)`.
